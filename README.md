Selenium Project (Python)
A small but meaningful step in my journey into test automation

Welcome to one of my very first Selenium projects.
This repository represents the early phase of my learning path — a place where I experimented, broke things, fixed them again, and slowly built a foundation for more advanced automation work.

It’s not meant to be a polished framework.
It’s meant to be honest.

🌱 Why I built this project
Before diving into complex architectures, CI pipelines, and full Page Object Models, I wanted to understand the basics properly:

How does Selenium WebDriver actually work

How to locate and interact with elements

How to handle waits and timing issues

How to structure tests in a clean, readable way

How to organize a simple automation project in GitHub

This repository was my sandbox — a safe place to learn by doing.

🧠 What I learned here
Working on this project taught me:

The fundamentals of browser automation

How to write stable locators

Why explicit waits matter

How to structure tests more modularly

How to debug flaky behavior

How to use GitHub to track progress and version my work

These lessons became the foundation for my next, much more advanced framework:

👉 Selenium_Project_Python_Pro  
(with POM, BasePage, Allure, CI/CD, logging, and full workflows)

🧱 Technologies used
Python

Selenium WebDriver

ChromeDriver

Basic PyTest usage

GitHub for version control

📂 What’s inside
This repository contains:

Simple Selenium scripts

Early attempts at structuring page interactions

Basic test flows

Experiments with locators, waits, and browser actions

It’s intentionally lightweight — the goal was learning, not perfection.

🚀 How this project evolved
This project was the starting point.
After understanding the basics here, I built a full professional automation framework:

👉 Selenium_Project_Python_Pro

That project includes:

Page Object Model

Allure reporting

GitHub Actions CI

Logging utilities

End‑to‑end workflows

This repository shows where the journey began.
